# サイト改善提案書

## 1. 現状の問題点

### 1.1 情報の階層構造が浅い

現状のページは全コンテンツが1ページに縦並びで配置されており、`<h2>` → `<h3>` → リスト という単純な階層しかない。セクション間に視覚的な区切りがなく、スクロールしていくと「今どのセクションを見ているのか」が分かりづらい。

### 1.2 セクションの粒度がばらばら

- 「興味の対象」「連絡先」は数行で終わる軽いセクション
- 「つくったもの・つくっているもの」は3Dモデル・シェーダー・ワールド・イベントを含む大きなセクション
- Twitter埋め込みがあるセクションは高さが極端に大きくなる

これらが同じレベルで並んでいるため、情報量の多いセクションに軽いセクションが埋もれる。

### 1.3 自己紹介と作品が混在している

ページ上部の「id → 近影 → 興味の対象 → 連絡先」という自己紹介パートと、「つくったもの」以降の作品紹介パートが同じ流れの中にあり、訪問者が「この人は誰か」と「何を作っているか」を区別しにくい。

### 1.4 ナビゲーションがない

ページ内リンクやメニューがないため、特定の情報（例：シェーダー作品だけ見たい）にたどり着くにはスクロールするしかない。

### 1.5 CSSがほぼ画像サイズ制御のみ

現在のCSSは4つのクラスに対して `width: auto; height: Npt;` を設定しているだけで、レイアウトやセクションの視覚的な区切りに関するスタイルがない。


## 2. 改善案

### 2.1 コンテンツを大カテゴリにグループ化する

現在のセクションを以下の3カテゴリに再編成する：

| カテゴリ | 含めるセクション |
|---------|--------------|
| **About** | id、近影、興味の対象、連絡先 |
| **Works** | 3Dモデル、シェーダー関連、VRChatワールド、イベント出展 |
| **Links** | Twitch、neort、BOOTH、ほしいもの |

### 2.2 ページ上部にナビゲーションを追加する

カテゴリレベルのページ内リンクを設置して、ワンクリックで目的のセクションに移動できるようにする。

```
例: [About] [Works] [Links]
```

### 2.3 カテゴリごとにセクションを視覚的に分離する

`<section>` タグでカテゴリを囲み、CSSで背景色・余白・区切り線などを付与してカテゴリの境界を明確にする。

```html
<nav>...</nav>
<section id="about">
  <!-- id, 近影, 興味の対象, 連絡先 -->
</section>
<section id="works">
  <!-- 3Dモデル, シェーダー, ワールド, イベント -->
</section>
<section id="links">
  <!-- 各種リンク, ほしいもの -->
</section>
```

### 2.4 Worksカテゴリ内をカード/タブ風に整理する

Works内は情報量が最も多い。サブカテゴリ（3Dモデル / シェーダー / ワールド / イベント）ごとにカード風のブロックで囲むと、作品同士の境界が明確になる。

### 2.5 CSSにレイアウト・セクション装飾を追加する

最低限追加すべきスタイル：

- `section` の余白・背景色の交互切り替え
- `nav` のスタイリング（固定ヘッダーまたはページ上部配置）
- カード/ブロックの枠線・影・角丸
- レスポンシブ対応（モバイルで画像がはみ出さないよう `max-width: 100%` など）

## 4. 実装の優先度

| 優先度 | 対応内容 | 理由 |
|-------|---------|------|
| 高 | セクションのグループ化 (`<section>` 導入) | 構造の基盤。他の改善がこれに依存する |
| 高 | ナビゲーション追加 | 少ないコード量で利便性が大きく向上する |
| 中 | CSSによるセクション装飾 | 視覚的な区切りで情報の散乱感を解消 |
| 中 | Works内のカード風レイアウト | 作品紹介の見やすさ向上 |
| 低 | レスポンシブ対応 | モバイル閲覧への対応 |
